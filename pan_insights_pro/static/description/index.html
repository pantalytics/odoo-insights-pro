See where your users get stuck — powered by PostHog

A lightweight, privacy-conscious asset module that injects PostHog into the Odoo backend. Dormant by default — it does nothing until you set a PostHog project key.

Dormant by default

No key means no script, no requests, no tracking. Set a project key when you're ready and reload — that's the whole setup.

Pseudonymous

Users are identified as db#uid — never a name or an email. Each database reports under its own PostHog workspace.

Privacy first

Input text is masked, autocapture ignores element values, and session replay is off. All toggleable, all on by default.

Zero friction

One backend asset, one settings page. No RPC round-trips: config rides along in the session on every page load.
